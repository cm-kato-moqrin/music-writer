# 進捗確認・記録

進捗の確認と記録を行います。

## 引数

なし（対話形式で進める）

## 手順

1. `progress/weekly-log.md` を読んで現在の状況を把握
2. `progress/skills-tracker.md` を確認
3. 該当フェーズの `plans/phase{N}/weekly-curriculum.md` を参照
4. ユーザーと対話して今週の振り返りを行う
5. 記録を更新

## 対話の流れ

### 1. 今週の振り返り

```
📊 第{N}週の振り返り

今週予定していた内容：
- {カリキュラムから抽出}

質問：
1. 上記の項目で達成できたものはどれですか？
2. 難しかったこと・課題はありましたか？
3. 新しく気づいたことはありますか？
```

### 2. スキルチェック

```
📋 スキルチェック

今週のチェック項目：
- [ ] {項目1}
- [ ] {項目2}
- [ ] {項目3}

どの項目にチェックを入れますか？
（自己評価でOKです）
```

### 3. 来週の目標設定

```
🎯 来週の目標

カリキュラムによると来週は：
- {次週の内容}

特に重点を置きたいことはありますか？
```

### 4. 記録の更新

ユーザーの回答に基づいて以下を更新：
- `progress/weekly-log.md` に今週の記録を追加
- `progress/skills-tracker.md` のチェックリストを更新

## weekly-log.md の形式

```markdown
## 第{N}週（{日付範囲}）

### 達成項目
- {達成した内容}

### 課題・難しかったこと
- {課題}

### 気づき・メモ
- {気づき}

### 来週への引き継ぎ
- {引き継ぎ事項}

---
```

## 注意

- ユーザーの自己評価を尊重する
- 遅れがあっても責めない、建設的な提案をする
- 小さな進歩も認める
- 必要に応じてカリキュラムの調整を提案する
